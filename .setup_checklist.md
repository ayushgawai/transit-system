# Setup Checklist

Use this checklist when setting up on a new machine:

## Pre-Setup

- [ ] Docker Desktop installed and running
- [ ] Python 3.9+ installed (`python3 --version`)
- [ ] Node.js 18+ installed (`node --version`)
- [ ] npm installed (`npm --version`)
- [ ] AWS CLI installed (`aws --version`)
- [ ] Git installed (`git --version`)

## Repository Setup

- [ ] Repository cloned
- [ ] Navigated to project directory
- [ ] All scripts are executable (`chmod +x *.sh`)

## AWS Configuration

- [ ] AWS CLI configured with profile `transit-system`
- [ ] Snowflake credentials stored in AWS Secrets Manager (`transit/snowflake-dev`)
- [ ] Perplexity API key stored in AWS Secrets Manager (`transit/perplexity`)
- [ ] OR local `secrets.yaml` file created (if not using AWS Secrets Manager)

## Configuration Files

- [ ] `config.yaml` exists and has `warehouse.type: 'snowflake'`
- [ ] `secrets.yaml` created (if using local secrets instead of AWS)

## First Run

- [ ] Run `./start_all.sh`
- [ ] Wait for all services to start (5-10 minutes first time)
- [ ] Frontend accessible at http://localhost:3000
- [ ] Backend API accessible at http://localhost:8000
- [ ] API Docs accessible at http://localhost:8000/docs
- [ ] Airflow UI accessible at http://localhost:8080 (admin/admin)

## Data Pipeline

- [ ] GTFS ingestion DAG triggered in Airflow
- [ ] Streaming DAG triggered in Airflow
- [ ] dbt models run successfully (`cd dbt/transit_dbt && dbt run --target snowflake`)
- [ ] ML forecast DAG triggered in Airflow

## Verification

- [ ] Dashboard shows data
- [ ] Live data page shows real-time departures
- [ ] Forecasts page shows ML predictions
- [ ] Chat interface works (LLM queries)

## Troubleshooting

If any step fails:
1. Check logs in `logs/` directory
2. Check Docker containers: `docker-compose -f docker-compose.local.yml --profile local ps`
3. Review SETUP.md troubleshooting section
4. Review README.md troubleshooting section

---

**Note**: This checklist assumes you're following the setup in SETUP.md
